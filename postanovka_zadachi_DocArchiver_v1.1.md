# Постановка задачи
## Система автоматизированного разбора и каталогизации электронного архива документов
### «DocArchiver» — Версия 1.1

---

## 1. Общие сведения

### 1.1. Наименование системы

- **Полное наименование:** Система автоматизированного разбора и каталогизации электронного архива документов
- **Краткое наименование:** DocArchiver

### 1.2. Назначение и цели создания

Цель разработки — создание программного инструмента, автоматизирующего трудоёмкий процесс ручного разбора большого архива электронных документов (книг). Система позволяет:

- в разы сократить время на инвентаризацию и классификацию архива;
- устранить дубликаты файлов, в том числе с разными именами и разными форматами;
- сформировать удобную иерархическую структуру хранения документов;
- автоматически экспортировать каталог документов в MS Excel.

### 1.3. Пользователи системы

Целевая аудитория — пользователь, управляющий личным или корпоративным архивом электронных книг и технических документов на ПК под управлением Windows 11. Специальных технических знаний не требуется.

### 1.4. Инструмент разработки

Программа разрабатывается с использованием **Claude Code** — AI-агента для разработки в терминале.

---

## 2. Описание предметной области

### 2.1. Характеристика архива

Архив представляет собой каталог на диске компьютера, содержащий:

- электронные книги в форматах PDF, EPUB, DJVU, FB2, DOCX и др.;
- произвольную вложенность подкаталогов (несколько уровней);
- смешанный состав: технические книги, учебники, справочники, художественная литература;
- потенциально большое количество файлов-дубликатов — в том числе **одна книга в нескольких форматах** (например, PDF + EPUB + FB2 одного издания).

### 2.2. Основные проблемы, требующие решения

| # | Проблема | Последствие |
|---|----------|-------------|
| 1 | Большой объём архива, ручной разбор невозможен | Архив не систематизирован |
| 2 | Отсутствие классификации документов | Сложно найти нужный документ |
| 3 | Дубликаты: одинаковые имена, разные форматы, разные имена | Избыточное использование дискового пространства |
| 4 | Хаотичная структура каталогов | Снижение эффективности работы с архивом |

## 3. Функциональные требования

### ФТ-1. Сканирование архива

**Описание:** Программа рекурсивно обходит указанный корневой каталог и формирует полный список документов.

**Детальные требования:**

- Пользователь выбирает корневой каталог через диалог выбора папки в GUI.
- Программа рекурсивно обходит все вложенные подкаталоги без ограничения глубины.
- Для каждого найденного файла фиксируются:
  - имя файла (с расширением);
  - полный путь к файлу (каталог расположения);
  - размер файла;
  - дата последнего изменения;
  - хэш-сумма содержимого (SHA-256) — для поиска точных дубликатов.
- Поддерживаемые форматы: PDF, EPUB, DJVU, FB2, DOCX, DOC, TXT, RTF, MOBI, AZW3.
- Ход сканирования отображается в прогресс-баре.

---

### ФТ-2. Классификация документов с помощью ИИ

**Описание:** Каждому документу автоматически присваивается класс (тематическая категория) и извлекаются метаданные.

**Детальные требования:**

- Классификация выполняется с использованием ИИ-модели (см. раздел 6).
- ИИ-модель анализирует для каждого документа:
  - имя файла;
  - метаданные документа (заголовок, автор, год, издательство — если доступны);
  - первые страницы / фрагмент текста (если извлечение технически возможно).
- ИИ возвращает структурированный JSON с полями:
  ```json
  {
    "title": "Наименование документа",
    "author": "Автор(ы)",
    "year": "Год издания или null",
    "publisher": "Издательство или null",
    "category": "Класс документа"
  }
  ```
- Перечень классов формируется автоматически, но пользователь может редактировать список вручную.
- Примеры классов:
  - Программирование → Python
  - Программирование → Web-разработка
  - Базы данных
  - Системное администрирование
  - Математика и статистика
  - Машинное обучение / ИИ
  - Управление проектами
  - Художественная литература
  - Прочее
- Пользователь может вручную изменить любое поле любого документа в интерфейсе.
- Финальный перечень классов экспортируется в MS Excel на отдельный лист.
- Промежуточные результаты кэшируются: повторный запуск не требует повторного обращения к ИИ.

---

### ФТ-3. Формирование каталога документов и экспорт в MS Excel

**Описание:** Программа формирует сводный каталог документов и выгружает его в файл MS Excel.

**Детальные требования:**

- Файл Excel содержит два листа:

  **Лист 1 — «Каталог документов»:**

  | Столбец | Описание |
  |---------|----------|
  | Имя файла | Имя файла с расширением |
  | Наименование документа | Название книги (из метаданных или ИИ) |
  | Автор(ы) | Имя автора или список авторов через «;» |
  | Год издания | Год издания или «—» если не определён |
  | Издательство | Название издательства или «—» если не определено |
  | Расположение (каталог) | Полный путь к папке с файлом |
  | Формат | Расширение файла (PDF, EPUB и т.д.) |
  | Размер (МБ) | Размер файла |
  | Класс документа | Тематическая категория |
  | Дубликат | Признак дубликата (Да/Нет) |
  | ID группы дубликатов | Идентификатор группы связанных файлов |

  **Лист 2 — «Классы документов»:**

  | Столбец | Описание |
  |---------|----------|
  | Код класса | Уникальный идентификатор |
  | Наименование класса | Название категории |
  | Количество документов | Число книг в категории |

- Файл Excel сохраняется в путь, указанный пользователем.
- Форматирование: заголовки выделены жирным, применена автоматическая ширина столбцов, строки с дубликатами подсвечены цветом.

---

### ФТ-4. Поиск и удаление дубликатов

**Описание:** Программа обнаруживает файлы с одинаковым содержимым, в том числе одну книгу в разных форматах.

**Детальные требования:**

- **Уровень 1 — точные дубликаты:** файлы с совпадающей SHA-256 хэш-суммой (100% идентичное содержимое).
- **Уровень 2 — форматные дубликаты:** одна книга в разных форматах (PDF + EPUB + FB2 одного и того же издания). Определяются ИИ-моделью по совпадению названия, автора и года.
- **Уровень 3 — смысловые дубликаты:** разные издания одной книги (при необходимости, опционально).

- В интерфейсе пользователь видит **список групп дубликатов**. Каждая группа содержит:
  - все версии/форматы книги;
  - предложение «Оставить» (рекомендуется наилучший формат или наибольший файл);
  - флажки для ручного управления.

- **Важно:** при выборе «оставить лучший формат» все форматы одной книги **копируются в одну папку** при реструктуризации (см. ФТ-5).

- Удаление дубликатов — **только после явного подтверждения** пользователем.
- Перед удалением создаётся лог-файл со списком удаляемых файлов.
- Опция: вместо удаления — перемещение в папку «Корзина архива».

---

### ФТ-5. Реструктуризация архива

**Описание:** Программа предлагает новую структуру хранения, создаёт каталоги и копирует/перемещает файлы.

**Детальные требования:**

- На основе классификации ИИ предлагает иерархическую структуру каталогов.
  Пример:
  ```
  Архив/
  ├── Программирование/
  │   ├── Python/
  │   │   ├── Лутц - Изучаем Python/        ← все форматы одной книги в одной папке
  │   │   │   ├── lutz_learning_python.pdf
  │   │   │   └── lutz_learning_python.epub
  │   │   └── Рамальхо - Python Fluently/
  │   │       └── fluent_python.pdf
  │   └── Web-разработка/
  ├── Базы данных/
  └── Художественная литература/
  ```

- **Ключевое правило:** все форматы одной книги (PDF, EPUB, FB2 и т.д.) помещаются в **одну подпапку**, названную по принципу `Автор - Название`.

- Пользователь может:
  - принять структуру целиком;
  - отредактировать её в интерфейсе (дерево каталогов);
  - переназначить отдельные книги в другие разделы.

- После **подтверждения** пользователем:
  - создаются новые каталоги на диске;
  - файлы копируются (или перемещаются — по выбору пользователя);
  - в Excel добавляется столбец «Новое расположение».

- Все операции протоколируются в лог-файл.

## 4. Нефункциональные требования

### 4.1. Операционная система

- **Основная:** Windows 11 (64-бит).
- **Желательно:** совместимость с Windows 10.

### 4.2. Технологический стек

| Компонент | Решение |
|-----------|---------|
| Язык разработки | Python 3.11+ |
| GUI-фреймворк | CustomTkinter или PyQt6 |
| Работа с Excel | openpyxl |
| Извлечение текста PDF | PyMuPDF (fitz) |
| Извлечение текста EPUB | ebooklib + BeautifulSoup4 |
| Извлечение метаданных | mutagen, PyMuPDF, ebooklib |
| Хэширование файлов | hashlib (стандартная библиотека) |
| ИИ-интеграция | google-generativeai, groq |
| Хранение настроек | python-dotenv (.env файл) |
| Кэш классификации | JSON-файл |
| Упаковка в EXE | PyInstaller |

### 4.3. Производительность

- Сканирование 10 000 файлов — не более 10 минут.
- Обращения к ИИ — пакетный режим с задержкой (rate limiting) для соблюдения лимитов API.
- Все длительные операции выполняются в отдельном потоке (без заморозки GUI).
- Кэш классификации исключает повторные запросы к ИИ при перезапуске программы.

### 4.4. Надёжность

- При ошибке доступа к файлу или сбое ИИ — программа продолжает работу, файл помечается «Не обработан», ошибка пишется в лог.
- Все деструктивные операции требуют явного подтверждения пользователя.

### 4.5. Удобство использования

- Интерфейс на русском языке.
- Прогресс-бар для длительных операций.
- Возможность приостановить и возобновить обработку.
- Экспорт лог-файла в TXT.

### 4.6. Дистрибуция

- Поставляется как единый исполняемый файл `.exe` (PyInstaller).
- Установка дополнительного ПО не требуется.

## 5. ИИ-модели: выбор и настройка

### 5.1. Актуальное состояние (май 2026)

> ⚠️ **Важно:** Gemini 1.5 Flash и все модели серии 1.x и 2.x **отключены**. Gemini 3.x — только платный доступ. Актуальные бесплатные модели — серия **Gemini 2.5**.

| Модель | Бесплатный лимит | Рекомендация |
|--------|-----------------|--------------|
| **Gemini 2.5 Flash** (`gemini-2.5-flash`) | ~15 RPM, ~500–1500 RPD | ⭐ Основная модель |
| **Gemini 2.5 Flash-Lite** (`gemini-2.5-flash-lite`) | ~30 RPM, ~1000 RPD | Резерв внутри Google |
| **Groq** (Llama 3.3, Mixtral) | 30 RPM, без лимита в день | ⭐ Резервная модель |
| Ollama (локально) | Без ограничений, офлайн | Для конфиденциальных данных |

> ⚠️ **Предупреждение по Gemini Free Tier:** Google значительно снизил лимиты в конце 2025 г. Реальные ограничения могут быть ниже официальных. Рекомендуется использовать **Groq** как основную рабочую модель при разборе большого архива, а Gemini — как приоритетную при наличии доступа.

### 5.2. Рекомендуемая схема переключения моделей

```
Попытка 1: Gemini 2.5 Flash
    → 429 Rate Limit или ошибка → Переключение на Groq
    → Groq недоступен → Попытка Gemini 2.5 Flash-Lite
    → Все недоступны → Запись в лог, продолжение с пустыми метаданными
```

Пользователь выбирает приоритет моделей в настройках программы.

#### Пример вызова Gemini 2.5 Flash

```python
import google.generativeai as genai
import os, json, time
from dotenv import load_dotenv

load_dotenv()
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))

model = genai.GenerativeModel("gemini-2.5-flash")

def classify_document(filename: str, text_excerpt: str) -> dict:
    """Классифицирует документ, возвращает dict с метаданными."""
    prompt = f"""
Ты — библиотечный каталогизатор. Проанализируй документ и верни ТОЛЬКО JSON (без markdown, без пояснений).

Имя файла: {filename}
Фрагмент текста (первые строки):
{text_excerpt[:1500]}

Верни строго в формате:
{{
  "title": "Название книги на языке оригинала",
  "author": "Автор или авторы через точку с запятой",
  "year": "Год издания или null",
  "publisher": "Издательство или null",
  "category": "Одна из категорий: Программирование/Python | Программирование/Web | Базы данных | Системное администрирование | Математика | Машинное обучение | Управление проектами | Художественная литература | Прочее"
}}
"""
    try:
        response = model.generate_content(prompt)
        raw = response.text.strip().removeprefix("```json").removesuffix("```").strip()
        return json.loads(raw)
    except json.JSONDecodeError:
        return {"title": filename, "author": None, "year": None, "publisher": None, "category": "Прочее"}
    except Exception as e:
        raise RuntimeError(f"Gemini API error: {e}")
```

#### Обработка Rate Limit (429)

```python
import time

def classify_with_retry(filename: str, text: str, max_retries: int = 3) -> dict:
    for attempt in range(max_retries):
        try:
            return classify_document(filename, text)
        except Exception as e:
            if "429" in str(e) or "quota" in str(e).lower():
                wait = 60 * (attempt + 1)  # 60с, 120с, 180с
                print(f"Rate limit Gemini. Ожидание {wait}с...")
                time.sleep(wait)
            else:
                raise
    # После исчерпания попыток — переключение на Groq
    return classify_via_groq(filename, text)
```

### 6.2. Groq API

#### Получение API-ключа

1. Перейти на [https://console.groq.com](https://console.groq.com)

#### Установка зависимостей

```bash
pip install groq
```

#### Пример вызова Groq (Llama 3.3 70B)

```python
from groq import Groq
import os, json
from dotenv import load_dotenv

load_dotenv()
groq_client = Groq(api_key=os.getenv("GROQ_API_KEY"))

# Актуальные модели Groq (май 2026):
# llama-3.3-70b-versatile  — рекомендуется
# mixtral-8x7b-32768        — резерв

def classify_via_groq(filename: str, text_excerpt: str) -> dict:
    prompt = f"""
Ты — библиотечный каталогизатор. Верни ТОЛЬКО JSON без markdown.

Файл: {filename}
Текст: {text_excerpt[:1500]}

Формат ответа:
{{"title": "...", "author": "...", "year": "...", "publisher": "...", "category": "..."}}
"""
    try:
        response = groq_client.chat.completions.create(
            model="llama-3.3-70b-versatile",
            messages=[{"role": "user", "content": prompt}],
            temperature=0.1,
            max_tokens=300,
        )
        raw = response.choices[0].message.content.strip()
        return json.loads(raw)
    except json.JSONDecodeError:
        return {"title": filename, "author": None, "year": None, "publisher": None, "category": "Прочее"}
    except Exception as e:
        raise RuntimeError(f"Groq API error: {e}")
```

### 6.3. Единый провайдер с автопереключением

```python
# ai/provider.py

class AIProvider:
    def __init__(self):
        self.primary = "gemini"    # или "groq"
        self._init_clients()

    def _init_clients(self):
        import google.generativeai as genai
        from groq import Groq
        import os

        gemini_key = os.getenv("GEMINI_API_KEY")
        groq_key = os.getenv("GROQ_API_KEY")

        self.gemini = None
        self.groq = None

        if gemini_key:
            genai.configure(api_key=gemini_key)
            self.gemini = genai.GenerativeModel("gemini-2.5-flash")

        if groq_key:
            self.groq = Groq(api_key=groq_key)

    def classify(self, filename: str, text: str) -> dict:
        """Классификация с автопереключением провайдеров."""
        providers = (
            [self._gemini_classify, self._groq_classify]
            if self.primary == "gemini"
            else [self._groq_classify, self._gemini_classify]
        )
        last_error = None
        for fn in providers:
            if fn is None:
                continue
            try:
                return fn(filename, text)
            except Exception as e:
                last_error = e
                continue
        # Оба провайдера недоступны
        return {"title": filename, "author": None, "year": None,
                "publisher": None, "category": "Прочее", "error": str(last_error)}
```

### 6.4. Кэширование результатов

```python
# utils/cache.py
import json, hashlib, os

CACHE_FILE = "cache/classifications.json"

def _file_key(filepath: str) -> str:
    return hashlib.md5(filepath.encode()).hexdigest()

def load_cache() -> dict:
    if os.path.exists(CACHE_FILE):
        with open(CACHE_FILE, "r", encoding="utf-8") as f:
            return json.load(f)
    return {}

def save_cache(cache: dict):
    os.makedirs("cache", exist_ok=True)
    with open(CACHE_FILE, "w", encoding="utf-8") as f:
        json.dump(cache, f, ensure_ascii=False, indent=2)

def get_cached(filepath: str, cache: dict) -> dict | None:
    return cache.get(_file_key(filepath))

def set_cached(filepath: str, result: dict, cache: dict):
    cache[_file_key(filepath)] = result
```

### 6.5. Настройки в GUI

В интерфейсе программы — раздел **«Настройки API»**, где пользователь:

- вводит GEMINI_API_KEY и GROQ_API_KEY (поля с маскировкой `•••`);
- выбирает приоритет: Gemini → Groq или Groq → Gemini;
- нажимает **«Проверить подключение»** — программа делает тестовый запрос и показывает статус;
- ключи сохраняются в `.env` файл рядом с программой.

> **Безопасность:** API-ключи хранятся в `.env` файле на локальном диске пользователя. По сети не передаются. В лог-файлы не записываются.

## 9. Ограничения и допущения

- Программа работает с локальной файловой системой Windows; сетевые диски поддерживаются при наличии прав.
- Для форматов DJVU и FB2 извлечение текста может быть ограничено; классификация тогда выполняется только по имени файла.
- Качество классификации зависит от доступности ИИ и качества метаданных файлов.
- Работа с Gemini и Groq требует подключения к интернету (Ollama — опциональный офлайн-вариант).
- API-ключи хранятся в `.env` файле локально — пользователь несёт ответственность за их безопасность.
- Лимиты Gemini Free Tier могут изменяться Google без предупреждения; рекомендуется иметь оба ключа (Gemini + Groq).
